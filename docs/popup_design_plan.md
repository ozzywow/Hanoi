## 0. 설계 원칙 (3줄 요약)
1. **의미(semantic) 우선**: 색·이모지는 "무슨 색"이 아니라 "무슨 역할"로 정의한다.
2. **코드로 강제**: 규칙은 문서로 두면 또 흐트러지므로, 공용 프레임 헬퍼 + 명명 컬러 상수로 박는다.
3. **톤으로 구분**: 성취/유틸/위험을 색으로 즉시 구분(골드=성취, 그레이=유틸, 레드=위험).

## 1. 컬러 팔레트 (지정 팔레트 밖 색상 사용 금지)

`common_define.h`에 명명 상수로 등록. 값은 현재 코드 기준 제안값(🟡).

### 1-1. 배경 / 프레임
| 이름 | 용도 | 제안값(RGB) | 상태 |
|------|------|------|------|
| `kPopupBG` | 팝업 배경 | 네이비 (10, 15, 50), α≈235 | 🟡 |
| `kBorderGold` | 성취/브랜드 창 테두리 | 골드 (255, 215, 0) | 🟡 |
| `kBorderGray` | 유틸/보조 창 테두리 | 슬레이트그레이 (150, 165, 185) | 🟡 |
| `kBorderRed` | 파괴적 위험 창 테두리 | 레드 (235, 90, 90) | ✅ (파괴적 확정 한정) |

**테두리 색 사용 기준 ✅ 확정:**
- 골드 = 성취·브랜드 창 (LEVEL CLEAR, NEW RECORD, 수상카드, 업데이트)
- 그레이 = 유틸·보조 창 (설정, 알림, 확인창, 이름입력)
- 레드 = **파괴적 위험 창 한정** (RESET ALL 등 되돌릴 수 없는 확정)
- ❓ 부정·비파괴 상태(YOU LOSE, TIME OUT)는 파괴가 아니므로 레드 프레임을 쓰지 않고,
  **골드/그레이 프레임 + 타이틀만 레드**로 톤을 표현하는 안 제안 → §6 논의 참조

### 1-2. 메시지 폰트 컬러 (의미별)
| 이름 | 역할 | 제안값 | 상태 |
|------|------|------|------|
| `kTextTitle` | 제목 전용 | 골드 (255, 215, 0) | 🟡 |
| `kTextData` | 핵심 데이터 (기록 시간, Player 이름) | 화이트 (255,255,255) | 🟡 |
| `kTextAccent` | 라벨/보조 강조 (LEVEL 3, 힌트) | 스카이블루 (120, 200, 255) | 🟡 |
| `kTextMuted` | 부가·캡션 | 그레이 (150, 160, 175) | 🟡 |
| `kTextError` | 에러 메시지 (이름 중복 등) | 레드 (255, 120, 120) | 🟡 |

**예외 — 상태값 컬러(데이터 표현용, 팔레트 규칙 적용 제외):**
RPM/승패 등 수치·상태 표시는 그린(80,220,80)/앰버(255,215,0)/레드(255,80,80) 3단계를 유지.

### 1-3. 텍스트(카피) 규칙 ✅
- **모든 UI 문구는 영어만 사용** — 제목·본문·라벨·버튼·힌트·에러 메시지 전부. (글로벌 타깃, 현지화 부담↓, 폰트/줄바꿈 일관)
- 한국어 문구 잔존분은 영어로 교체 (예: TIME OUT 본문, RESET 확인 본문 등).
- **예외 — 유저 생성 콘텐츠(UGC)**: 유저 이름, 수상소감 등 유저가 입력한 텍스트는 원문 언어 그대로 표시(강제 영어 아님).

## 3. 버튼 디자인 규칙

### 3-1. 역할 → 컬러 → 글리프 (고정 매핑)
버튼 폰트색 = 테두리색 (일치). 배경은 해당색의 어두운 반투명 채움(칩).

| 역할 | 테두리·폰트색 | 글리프 | 해당 버튼 |
|------|------|------|------|
| 긍정·실행 | 스카이블루 `kBtnFunc` (120,200,255) | ▶ / ✓ / ↑ / ✎ / 👁 | REPLAY, WATCH, OK, POST, UPDATE, EDIT |
| 경쟁·공격 | 오렌지 `kBtnRace` (255,180,90) | ⚔ / ↻ | RACE, REVENGE, RETRY |
| 해제·취소 | 그레이 `kBtnDismiss` (170,178,195) | ✕ | CLOSE, CANCEL, LATER |
| 파괴·확정 | 레드 `kBtnDanger` (235,90,90) | ⚠ | RESET 확정 |

### 3-2. 이모지 vs 심볼 ✅ 확정
- ✅ **단색 심볼 글리프로 통일**(▶ ⚔ ↻ ✕ ✓ ↑ ✎)해 라벨 색을 그대로 상속(폰트색=테두리색 규칙 준수).
- ✅ 컬러 이모지(🚩🏠 등)는 **가능하면 단색 심볼로 대체**, 마땅한 심볼이 없을 때만 예외적으로 컬러 이모지 허용.
  - 적용 대상: 🚩 Report → **⚑**, 🏠 Home → **⌂**, 🕐 Later → 심볼 부재 → **텍스트만**(글리프 생략).
  - **예외 허용: 👁 WATCH(관전)** — 관전을 뜻하는 마땅한 단색 심볼이 없어 컬러 이모지 유지(테두리색과 불일치 감수).

### 3-3. 버튼 지오메트리 (칩 버튼, 이미 부분 통일됨)
| 항목 | 규칙 |
|------|------|
| 형태 | `makePopupChipButton` 칩(테두리+반투명 채움), 히트영역 확대 |
| **하단 버튼 Y** | **`kPopupBtnY`(=26, 박스 바닥 기준) 공용 상수로 통일** — 팝업마다 제각각 금지. 칩 하단 여백 ~8px |
| 2버튼 배치 | 좌 x=74 / 우 x=(PW−74) |
| **배치 의미** | **주 액션 = 좌 / 취소·닫기 = 우** (파괴창 RESET은 기본 무게를 우측 Cancel에) |
| 단일 버튼 | 중앙 x=PW/2 |
| 크기 | 표준 칩 112×38 (좁은 창은 축소 허용) |

## 6. 결정 사항

### ✅ 확정됨 (2026-07-15)
1. **레드 톤 도입** — 단, **파괴적 확정(RESET ALL 등 되돌릴 수 없는 액션)에 한정** 사용.
2. **이모티 = 단색 심볼로 통일** — 컬러 이모지는 가능하면 단색 심볼 대체, 불가 시에만 예외 허용.
3. **테두리 배분** — 성취=골드 / 유틸=그레이 / 파괴적 위험=레드.

### ✅ 확정됨 (2026-07-15, 2차)
A. **부정·비파괴 상태 톤** (YOU LOSE, TIME OUT) — **프레임=그레이 + 타이틀만 레드**.
   (YOU LOSE도 성취 골드가 아니라 **그레이 프레임**으로 확정 — 패배는 성취창 아님.)
B. **버튼 좌우 배치** — **주 액션 = 좌 / 취소·닫기 = 우** 고정.
   단 파괴적 확정창(RESET)은 기본 무게를 안전한 Cancel 쪽에 두는 예외 허용.
C. **모달 처리 일원화** — 배경 터치 차단을 `attachModalBlocker` 방식으로 통일
   (결과 팝업의 오버레이 스왈로우도 이 방식으로 흡수).
D. **중복 글리프** — **👁 WATCH / ▶ REPLAY 구분 사용** (2026-07-15 변경, 종전 "둘 다 ▶"에서 수정).
   RACE/REVENGE는 둘 다 ⚔(텍스트로 구분).
   ※ 👁는 마땅한 단색 심볼이 없어 **규칙 3-2 예외(컬러 이모지 허용)** 로 처리 → 스카이블루 테두리색과
     완전 일치는 불가(멀티컬러 눈). 이 점 감수하고 관전=👁로 구분 우선.
E. **TIME OUT 배경** — 적갈색 → **네이비 강제 통일**. "나쁨"은 타이틀 레드로만 표현.

F. **백드롭 딤 불투명도** ✅ — **표준 160 통일 + 강제 업데이트만 210 예외**.

G. **REPLAY 라벨** ✅ — 라벨은 **의미 구분 유지**: `▶ REPLAY`(내 기록 다시보기) / `👁 WATCH`(랭커 관전). 글리프도 D에 따라 구분.

> **🎯 모든 규칙 확정 완료 (2026-07-15).** 남은 ❓ 없음. 팔레트 RGB 세부값(🟡)만 구현 시 미세조정 가능.
> 다음 단계: `common_define.h` 컬러 명명 상수 → `DrawUtils` 공용 프레임 헬퍼 → 각 팝업 리팩터.

## 7. 구현 방향 (확정 후 진행 — 참고용)
- `common_define.h`: 위 명명 컬러 상수 등록(팔레트 이탈 원천 차단).
- `DrawUtils`: 공용 프레임 헬퍼 `makePopupFrame(title, tone, size)` — 백드롭+박스+테두리+타이틀+구분선+스케일인 표준 생성. 버튼은 기존 `makePopupChipButton`에 role 파라미터 추가.
- 각 팝업 함수는 프레임 헬퍼 + 칩 버튼만 조립하도록 리팩터.
